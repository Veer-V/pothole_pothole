1.  System Overview & Architecture 
 
 
This blueprint describes a full production-grade pothole detection system using a Raspberry Pi 4 as 
the edge brain, a TF-Luna LiDAR as the primary close-range depth sensor, a Benewake TF02-i 
CAN LiDAR as the long-range / high-precision secondary sensor, and an HC-SR04 ultrasonic as an 
automatic fallback. All three sensors feed a sensor-fusion layer that picks the most reliable reading 
at every moment. Detected potholes — with GPS coordinates, measured depth, estimated width, 
and AI severity score — are streamed over MQTT to a cloud backend and displayed on a live 
government web dashboard.

4.  Edge Firmware — Sensor Code 
 
 
4.1  TF-Luna Driver (UART Mode) 
 
# tf_luna.py — TF-Luna UART driver for Raspberry Pi 
import serial, struct, time 
 
class TFLuna: 
    HEADER = 0x59  # TF-Luna frame header byte 
 
    def __init__(self, port='/dev/ttyS0', baudrate=115200): 
        self.ser = serial.Serial(port, baudrate, timeout=0.1) 
        self.last_valid = None 
 
    def read_distance(self): 
        """ Returns distance in cm, or None on error """ 
        try: 
            # Sync to frame start (two consecutive 0x59 bytes) 
            while True: 
                if self.ser.read(1)[0] == self.HEADER: 
                    if self.ser.read(1)[0] == self.HEADER: 
                        break 
            frame = self.ser.read(7)  # read remaining 7 bytes 
            dist_low, dist_high = frame[0], frame[1] 
            distance_cm = (dist_high << 8) | dist_low 
            # Validate checksum 
            checksum = (self.HEADER * 2 + sum(frame[:8])) & 0xFF 
            if checksum != frame[8] if len(frame) > 8 else True: 
                return None 
            self.last_valid = distance_cm 
            return distance_cm 
        except Exception: 
            return None  # Caller will trigger fallback 
 
    def is_healthy(self): 
        return self.read_distance() is not None

    def read_distance(self): 
        """ Returns distance in cm, or None on error """ 
        try: 
            msg = self.bus.recv(timeout=0.05)  # 50ms timeout 
            if msg and msg.arbitration_id == TF02I_CAN_ID: 
                # TF02-i frame: bytes 0-1 = distance (mm), little-endian 
                dist_mm = struct.unpack_from('<H', msg.data, 0)[0] 
                dist_cm = dist_mm / 10.0 
                self.last_valid = dist_cm 
                return dist_cm 
        except Exception: 
            return None 
        return None 
 
    def is_healthy(self): 
        return self.read_distance() is not None 
 
    def shutdown(self): 
        self.bus.shutdown() 
 
4.3  HC-SR04 Backup Ultrasonic Driver 
 
# hcsr04.py — HC-SR04 fallback depth sensor 
import RPi.GPIO as GPIO, time 
 
TRIG, ECHO = 23, 24 
BASELINE_CM = 25.0  # flat road calibration distance (set during install) 
 
GPIO.setmode(GPIO.BCM) 
GPIO.setup(TRIG, GPIO.OUT) 
GPIO.setup(ECHO, GPIO.IN) 
 
def get_raw_distance(): 
    GPIO.output(TRIG, True) 
    time.sleep(0.00001) 
    GPIO.output(TRIG, False) 
    timeout = time.time() + 0.04  # 40ms max 
    while GPIO.input(ECHO) == 0: 
        pulse_start = time.time() 
        if time.time() > timeout: return None 
    while GPIO.input(ECHO) == 1: 
        pulse_end = time.time() 
        if time.time() > timeout: return None 
    return ((pulse_end - pulse_start) * 34300) / 2 
 
def get_depth_hcsr04(): 
    raw = get_raw_distance() 
    if raw is None: return None 
    depth = raw - BASELINE_CM 
    return max(0.0, round(depth, 1))

This is the key module that makes the dual-LiDAR + backup system work. It tries TF-Luna first, then 
TF02-i, fuses both when available, and silently falls back to HC-SR04: 
 
# depth_fusion.py — Master sensor fusion depth module 
from tf_luna import TFLuna 
from tf02i_can import TF02iCAN 
from hcsr04 import get_depth_hcsr04 
import logging 
 
# Fusion weights: TF-Luna higher weight (better short-range accuracy) 
WEIGHT_LUNA = 0.60 
WEIGHT_TF02 = 0.40 
 
luna  = TFLuna(port='/dev/ttyS0') 
tf02i = TF02iCAN(channel='can0') 
 
def get_fused_depth(): 
    """ 
    Returns: (depth_cm: float, source: str) 
    source: 'lidar_fused' | 'lidar_luna' | 'lidar_tf02' | 'ultrasonic_backup' 
    """ 
    d_luna = luna.read_distance() 
    d_tf02 = tf02i.read_distance() 
 
    if d_luna is not None and d_tf02 is not None: 
        # CASE 1: Both LiDARs healthy — weighted fusion 
        fused = (WEIGHT_LUNA * d_luna) + (WEIGHT_TF02 * d_tf02) 
        return round(fused, 2), 'lidar_fused' 
 
    elif d_luna is not None: 
        # CASE 2: Only TF-Luna available 
        return round(d_luna, 2), 'lidar_luna' 
 
    elif d_tf02 is not None: 
        # CASE 3: Only TF02-i available 
        return round(d_tf02, 2), 'lidar_tf02' 
 
    else: 
        # CASE 4: Both LiDARs failed — use HC-SR04 backup 
        logging.warning('LiDAR offline — switching to HC-SR04 backup') 
        depth = get_depth_hcsr04() 
        return (depth or 0.0), 'ultrasonic_backup'

5.  IMU Trigger & Main Orchestration Loop 
 
 
5.1  Pothole Trigger (MPU-6050) 
 
# imu_trigger.py 
import smbus2, math, time 
 
MPU_ADDR = 0x68 
THRESHOLD_G = 2.5   # tune per vehicle type (bike ~2.5, car ~3.0) 
COOLDOWN_S  = 1.5   # ignore re-triggers within 1.5s (same pothole) 
 
bus = smbus2.SMBus(1) 
bus.write_byte_data(MPU_ADDR, 0x6B, 0)  # wake up MPU-6050 
 
_last_trigger = 0 
 
def read_accel(): 
    raw = bus.read_i2c_block_data(MPU_ADDR, 0x3B, 6) 
    ax = ((raw[0]<<8)|raw[1]) / 16384.0 
    ay = ((raw[2]<<8)|raw[3]) / 16384.0 
    az = ((raw[4]<<8)|raw[5]) / 16384.0 
    return ax, ay, az 
 
def check_trigger(): 
    global _last_trigger 
    ax, ay, az = read_accel() 
    jolt = abs(az - 1.0)  # 1g = normal gravity, subtract baseline 
    if jolt > THRESHOLD_G: 
        now = time.time() 
        if now - _last_trigger > COOLDOWN_S: 
            _last_trigger = now 
            return True, round(jolt, 3) 
    return False, 0.0

10.  Sensor Calibration Guide 
 
 
10.1  TF-Luna Calibration 
•​
Mount sensor pointing straight down at road surface, firmly fixed (no vibration play) 
•​
Park on flat road: record 100 readings at known height. Average = your BASELINE_CM 
•​
Use Benewake TF-Luna GUI tool (Windows) via USB-TTL to verify frame output and change 
frame rate if needed 
•​
Default 250Hz is ideal — do not reduce below 100Hz for pothole detection 
•​
Valid range check: discard readings < 20cm (too close) or > 600cm (too far = bad reflection) 
 
10.2  TF02-i CAN Calibration 
•​
Use TF02-i config tool or candump to read raw CAN frames first 
•​
TF02-i default CAN ID: 0x700 — confirm with your specific unit's datasheet 
•​
Distance is reported in millimeters in bytes 0–1 (little-endian uint16) 
•​
Park on flat road: verify TF02-i and TF-Luna give matching readings ±5mm 
•​
If deviation > 10mm, check sensor mounting angle — must be perfectly perpendicular to 
road 
 
10.3  Fusion Validation Test 
1.​ Cover TF-Luna with hand → system should switch to 'lidar_tf02' mode only 
2.​ Cover TF02-i CAN → system should switch to 'lidar_luna' mode only 
3.​ Cover both → system switches to 'ultrasonic_backup', dashboard shows ⚠️ alert 
4.​ Uncover both → system auto-returns to 'lidar_fused' mode within 2 seconds 
 
10.4  Camera Focal Length Calibration 
•​
Print a 30cm × 30cm white square on paper. Tape flat to ground. 
•​
Position sensor rig exactly 100cm above it (use measuring tape). 
•​
Run camera, detect the square. Count pixel width of bounding box. 
•​
FOCAL_PX = (pixel_width × 100) / 30 
•​
Example: if bbox is 200px wide → FOCAL_PX = (200 × 100) / 30 = 667
